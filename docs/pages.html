Frontend Pages

App Pages Overview

Blockchella's frontend is a set of plain HTML pages, each served statically with no framework. All blockchain interactions happen in the browser via ethers.js and MetaMask. Below is every page, who uses it, and what it does.

landing.html — Role Selector

All Users No wallet needed

The first page users see. Presents three role cards: Attendee, Doorman, and Venue Admin. Clicking a card navigates to the appropriate entry point for that role. No MetaMask connection is required here.

home.html — Attendee Home

Attendee

The main landing page for attendees. Explains the Blockchella concept with a hero section, shows key stats pulled from the contract (tickets available, total sold), and links to all attendee actions. Also includes a live connection status bar.

get-started.html — Setup Guide

Attendee No wallet needed

An in-app walkthrough for first-time users. Covers installing MetaMask, switching to Sepolia, and getting SETH from a faucet. Includes a wallet connect section and a connection status check to confirm setup is complete.

group.html — Create Group

Attendee (Group Creator) Requires wallet

Lets a connected attendee create a new group by entering their friends' wallet addresses and a deadline in hours. On submission, calls createGroup(members, deadlineHours). After creation, the page shows the generated Group ID which must be shared with all group members so they can join and pay.

Key features: dynamic friend address fields, total cost preview, live Group ID display after creation.

buy.html — Buy Ticket

Attendee Requires wallet to purchase

Two-mode purchase page with a toggle between Buy Solo and Join a Group.

  • Buy Solo — Creates a single-member group and immediately purchases the ticket. Ticket price and availability are loaded without needing MetaMask.
  • Join a Group — Enter a Group ID to load group details (member list, deadline countdown, payment status) then call joinGroup to pay into escrow.

balance.html — My Ticket

Attendee Requires wallet

Shows the connected wallet's ticket status, SETH balance, and group membership details. Displays a colour-coded ticket status (green = ticket held, red = no ticket). If the wallet is in an active group, shows the group countdown timer and member payment progress. Includes a Refresh button and real-time updates.

verify.html — Ticket Verification

Doorman No wallet needed

For use by event door staff. Enter any wallet address to check whether it holds a valid ticket token. Displays a large ✓ VALID or ✗ INVALID result. No MetaMask required — uses a read-only provider. This page is not linked from the attendee navigation to keep role separation clear.

return.html — Return Ticket

Attendee (Ticket Holder) Requires wallet

Allows a ticket holder to return their ticket for a full refund of the ticket price. Shows current balances, refund amount, and a confirmation checkbox before enabling the return button. Also includes an "Unclaimed Refunds" section to claim any failed push refunds via claimRefund().

admin.html — Venue Admin Panel

Venue (Contract Owner Only) Requires owner wallet

Full admin dashboard for the contract owner. Shows a stats grid (tickets sold, SETH in escrow, active/completed/cancelled groups), a paginated table of all groups, and the full Blockchella chain explorer with block hash verification. Includes a Withdraw Funds button and real-time updates toggle.

Connecting with any wallet other than the contract owner address will show an ownership mismatch warning.

wallet.html — Wallet Setup

Attendee No wallet needed

A dedicated guide page helping new users set up MetaMask, switch to Sepolia, and understand what a wallet is. Linked from the attendee navigation for users who don't know where to start. Purely informational — no contract interaction.
